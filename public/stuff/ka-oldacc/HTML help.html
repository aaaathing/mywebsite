HTML <!DOCTYPE> Declaration

A doctype is not a html element or tag. It is information for the browser to tell it what type of document the rest of the page will be.

You should always do this in a HTML5 webpage:

<!DOCTYPE html>

Sometimes, you won't use HTML5. You might use some older HTML version.
In that case, you will have a more complicated doctype.

HTML 4.1
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
"http://www.w3.org/TR/html4/strict.dtd">

XHTML 1.1
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN"
"http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">

Where to put a doctype

A doctype needs to go before all the html.
That means it can be on any line that comes before the html on the page.

<!DOCTYPE html>
<html>
  <head>
    <title>Webpage</title>
  </head>
  <body>
    Content goes here
  </body>
</html>

Spelling

A doctype can be uppercase or lowercase or a mix of both

<!DOCTYPE html> Recomendded

<!doctype html> Allowed

<!DoCtYpE html> Allowed

HTML <a> tag

The HTML <a> tag is used to create links in html.
To set the link, use the href attribute.

<a href="https://website.com">Link Text</a>

Attributes

Attribute Required Value Description
href Required URL This attribute specifies where the link should lead to.
target Optional _blank
_parent
_self
_top		 Specifies where to open the linked document
download Optional filename Specifies that the target will be downloaded when a user clicks on the hyperlink

HTML <base> tag

The <base> tag specifies the base URL and/or target for all relative URLs in a document.

The <base>; tag must have either an href or a target attribute present, or both.

There can only be one single <base> element in a document, and it must be inside the <head> element.

Specify a default URL and a default target for all links on a page:

<head>
  <base href="https://www.w3schools.com">
</head>
<body>
  <image src="images/stickman.gif" width="240" height="39" alt="Stickman">
  <a href="tags/tag_base.asp">HTML base Tag</a>
</body>

HTML <article> tag

The <article> tag specifies independent, self-contained content.

An article should make sense on its own and it should be possible to distribute it independently from the rest of the site.

Potential sources for the <article> element:

  • Forum post
  • Blog post
  • News story

Note: The <article> element does not render as anything special in a browser.

Example:

<article>
  <h2>Google Chrome</h2>
  <p>Google Chrome is a web browser developed by Google, released in 2008. Chrome is the world's most popular web browser today!</p> </article>

<article>
  <h2>Mozilla Firefox</h2>
  <p>Mozilla Firefox is an open-source web browser developed by Mozilla. Firefox has been the second most popular web browser since January, 2018.</p>
</article>

<article>
  <h2>Microsoft Edge</h2>
  <p>Microsoft Edge is a web browser developed by Microsoft, released in 2015. Microsoft Edge replaced Internet Explorer.</p>
</article>

HTML <blockquote> tag

The <blockquote> tag specifies a section that is quoted from another source.
Browsers usually indent <blockquote> elements.

<blockquote cite="http://www.worldwildlife.org/who/index.html">
For 50 years, WWF has been protecting the future of nature. The world's leading conservation organization, WWF works in 100 countries and is supported by 1.2 million members in the United States and close to 5 million globally.
</blockquote>

Attributes

Attribute Required Value Description
cite Required URL Specifies the source of the quotation
